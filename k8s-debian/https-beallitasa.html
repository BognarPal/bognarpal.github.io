Https beállítása

Cert manager telepítése

Feladat, hogy a https certifikációt időről időre frissítse

helm repo add jetstack https://charts.jetstack.io
helm repo update
A telepítés: 
helm install \
cert-manager jetstack/cert-manager \
--namespace cert-manager \
--create-namespace \
--version v1.9.1 \
--set installCRDs=true

Config fájlok

issuer.yaml

apiVersion: cert-manager.io/v1
kind: ClusterIssuer
metadata:
  name: letsencrypt-prod
spec:
  acme:
    server: https://acme-v02.api.letsencrypt.org/directory
    email: "bognar.pal@jedlik.eu"
    privateKeySecretRef:
      name: letsencrypt-prod
    solvers:
      - http01:
          ingress:
            class: nginx

certificate.yaml

apiVersion: cert-manager.io/v1

kind: Certificate
metadata:
  name: cluster-jedlik-eu-tls
spec:
  secretName: cluster-jedlik-eu
  issuerRef:
    name: letsencrypt-prod
    kind: ClusterIssuer
  commonName: cluster.jedlik.eu
  dnsNames:
    - cluster.jedlik.eu

Config fájlok futtatása

kubectl apply -f issuer.yaml
kubectl apply -f certificate.yaml

Ingres config módosítások

Két új "annotation"

    cert-manager.io/cluster-issuer: 'letsencrypt-prod'
    nginx.ingress.kubernetes.io/ssl-redirect: 'true'

A spec részben (a rules fölé) új beállítás

  tls:
    - hosts:
        - cluster.jedlik.eu
      secretName: cluster-jedlik-eu
