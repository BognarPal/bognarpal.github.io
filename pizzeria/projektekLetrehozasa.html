Projektek létrehozása

A feladat definiálása során meghatározott feladatokhoz - szerintem - az alábbi projekteket kellene létrehozni a solutionben:

  • pizzeria.data: ez a projekt felelne az adatelérésért
  • pizzeria.service: ebben a rétegben kerülne definiálsra az üzleti logika
  • pizzeria.website: ez a projekt felelne a webes megjelenésért (Rest API & Angular)

A fenti videóban (Windows-os verzió) Visual Studó 2022-et használtam (a Preview 2.0-s verziót), és a keretrendszernek a .Net Core 6.0 (szintén még csak preview verziót) választottam. A projekteket létrehozása után a két class library-ből töröltem a Class1.cs állományokat.

A pizzeria.website projekt típusa ASP.NET Core Web App, amihez integrálva van egy Angular alkalmazás is. Ezen dokumentum készítésekor ennek az Angular alkalmazásnak a verziószáma 12.0 volt, miközben az Angular hivatalos oldalán a legfrisebb elérhető verzió a 12.1-es. Ekkora eltérés nem lenne probléma, azonban a tapasztalat az, hogy a Microsoft nem követi ennél a projekt típusnál az Angular verzió frissítéseit.

A fentiek miatt érdemes elvégezni a website projekt könyvtárában lévő ClientApp mappában lévő Angular alkalmazás frissítését.

További (később) elvégzendő feladatok:

  • a webalkalmazásból törölni kell az időjárás előrejelzéssel kapcsolatos tartalmakat.
  • a létrehozott projektek közötti referenciákat be kell állítani

Projekt első indítása

Mivel még az időjárás jelentésre vonatkozó részek törlése nem történt meg, ezért a projekt indítása után ez fog megjelenni.

Az első indítás a számítógép sebességétől függően néhány percig is eltarthat, hiszen az Angular kliensnek le kell töltenie a szükséges csomagokat.

A Visual Studio-ban egy CTRL-F5-t nyomva megtörténik mind a kliens (Angular), mind a szerver oldali (C#) kódok fordítása. Az így létrejött futtatható programot a Visual Studio 2019-től eltérő módon már nem az IIS Express, mint webszerver fogja kiszolgálni, hanem a backend futtatását .Net CLI, a frontend futtatását az Angular Cli végzi. (Amennyiben szükség lenne a kód debug-golására akkor, F5-el is indítható.)

A sikeres indítást követően az alapértelmezett böngészőben megjelenik a weboldal. Elképzelhető hogy engedélyezni kell a https megjelenítését, ugyanis a használt tanusítvány nincs digitálisan aláírva.

Terminál ablakból a megfelelő könyvtárban dotnet run paranccsal lehet az alkalmazást elindítani. A videóban én a Visual Studio Code beépített terminál ablakát használtam. A böngésző nem indul el automatikusan, azt kézzel kell indítani és a címet beírni.
